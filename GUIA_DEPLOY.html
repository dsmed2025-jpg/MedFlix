Pinguim Filmes — Guia de Deploy

Como publicar seu app Flask na internet de graça (ou quase)

⚠️
Por que o Netlify não funciona?
O Netlify só serve sites estáticos (HTML/CSS/JS puro). O Pinguim Filmes é uma aplicação Python/Flask com banco SQLite, sessões e rotas dinâmicas — precisa de um servidor real que execute Python.

🏆 Melhores opções gratuitas para Flask

✓ Recomendado
Render.com
Grátis (com limitações) / $7/mês
Deploy automático do GitHub. Suporte a disco persistente para o SQLite. Roda Flask/gunicorn sem configuração extra. Plano grátis "dorme" após 15 min de inatividade.
Railway.app
$5/mês (plano mínimo pago)
Muito simples, detecção automática de Python. Banco de dados persistente. Não tem plano 100% grátis, mas $5/mês garante uptime contínuo.
Fly.io
Grátis até certos limites
Servidor em São Paulo (GRU), baixa latência no Brasil. Requer CLI e cartão de crédito para verificação. Mais técnico que os outros.
VPS (Hostinger, etc.)
~R$25/mês
Controle total, sem limitações. Instala Python/Flask direto no servidor Linux. Ideal para produção real com muitos usuários.

🚀 Deploy no Render.com (mais fácil)

Todos os arquivos de configuração já estão incluídos no ZIP (render.yaml, Procfile, requirements.txt).

1
Crie uma conta no GitHub (se não tiver) Acesse github.com → "Sign up" → crie sua conta gratuita
2
Crie um repositório e suba o projeto No GitHub: clique em "New repository" → nomeie como pinguim-filmes → crie. Depois faça upload de todos os arquivos do ZIP (exceto a pasta __pycache__).
3
Crie conta no Render.com Acesse render.com → "Get Started for Free" → conecte com sua conta GitHub
4
Novo Web Service No painel do Render: clique em "New +" → "Web Service" → selecione seu repositório pinguim-filmes
5
Configure os campos O Render detecta tudo automaticamente pelo render.yaml. Confirme as configurações abaixo e clique em "Create Web Service":
Name: pinguim-filmes Runtime: Python 3 Build Command: pip install -r requirements.txt Start Command: gunicorn app:app --workers 2 --timeout 120 --bind 0.0.0.0:$PORT
6
Adicione as variáveis de ambiente Em "Environment" → "Add Environment Variable":
VariávelValorDescrição
TMDB_API_KEY0cc6bfd91cbdb1775b79fd4267b9e3c2Chave TMDB para capas e trailers
TMDB_READ_TOKENeyJhbGci... (do .env)Token de leitura TMDB
PINGUIM_SECRET_KEYQualquer texto aleatório longoSegurança das sessões
7
Adicione disco persistente (importante!) Em "Disks" → "Add Disk" → nome: pinguim-db → Mount Path: /opt/render/project/src → Size: 1 GB. Isso garante que o banco SQLite não seja apagado a cada deploy.
✅ Após ~3 minutos de build, seu app estará em https://pinguim-filmes.onrender.com
⚠️ No plano gratuito do Render, o app "dorme" após 15 minutos sem acesso. O primeiro acesso demora ~30 segundos para "acordar". Upgrade para $7/mês elimina isso.
Acessar Render.com →

🚂 Deploy no Railway.app (mais simples ainda)

1
Acesse railway.app e conecte o GitHub railway.app → "Start a New Project" → "Deploy from GitHub repo"
2
Selecione o repositório pinguim-filmes O Railway detecta Python automaticamente pelo railway.json e nixpacks.toml incluídos no projeto.
3
Adicione as variáveis de ambiente Clique no serviço → "Variables" → adicione as 3 variáveis da tabela acima (TMDB_API_KEY, TMDB_READ_TOKEN, PINGUIM_SECRET_KEY)
4
Gere o domínio público Em "Settings" → "Networking" → "Generate Domain" → seu app fica em pinguim-filmes.up.railway.app
Acessar Railway.app →

🖥️ VPS (Produção real — R$25/mês)

Para quem quer controle total, sem limitações de "modo sleep" e múltiplos usuários simultâneos. Use a Hostinger, DigitalOcean, ou qualquer VPS Ubuntu.
